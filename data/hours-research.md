# Opening-hours research — London (KY location list)

Research pass to attach **opening hours** to the catalogue's London spots, so they
can be surfaced in the app. Generated by `tools/enrich-hours.js`; the data lives in
[`data/hours.json`](./hours.json) as a **separate sidecar keyed by spot id** —
`data/spots.json` and every writeup are untouched.

## Scope
- **Hours-relevant categories only** (456 of 996 London spots). The other 540 —
  statues, memorials, mosaics, wells, parks, streets, ghost signs, plaques — are
  outdoor / always-open / no fixed hours, so they were deliberately skipped.
- The 23 categories researched: `pub, food, museum, brunch, bakery, coffee, caff,
  cinema, boba, pieandmash, matcha, wine, bookshops, vinyl, dumpling, music,
  diaspora, lido, archive, view, stadium, livery, polish`.

## Method
- Source: **OpenStreetMap's structured `opening_hours` tag** (ODbL), pulled per spot
  via Overpass (`around:200m`), then matched to the catalogue **by name** — exact,
  containment, or strong token overlap. A wide 200 m net is used because a spot's
  recorded coordinate often sits 100–200 m from the building centroid OSM tags the
  hours on; a same-named element that far away is still almost certainly the place.
- **Precision over recall, on purpose.** An earlier pass also accepted the *nearest*
  hours-tagged element when no name matched — that grabbed neighbours (a café's hours
  became the *Apple Store* next door; a caff's became the *Sainsbury's*). All 33 such
  "near" guesses were dropped and the fallback removed. A missing entry means
  "unknown", **never a guessed value**.
- Web spot-check of a sample (below) to gauge how trustworthy the OSM values are.

## Results — 109 / 456 matched (24%)

Coverage is very uneven. Established, well-known venues are well mapped; newer/trendy
spots and "lore" landmarks (band sites, ex-clubs) mostly aren't in OSM with hours.

## How trustworthy is it? (web spot-check)

| Spot | OSM value | Web (current) | Verdict |
|---|---|---|---|
| Florence Nightingale Museum | Tu–Su 10:00–17:00; late last Thu | identical | ✅ exact |
| Grant Museum of Zoology | Tu–Fr 13–17, Sa 11–17 | identical | ✅ exact |
| The Mayflower (pub) | Su 12:00–22:00 | Su 12:00–**21:00** | ⚠️ 1 h stale |
| Quality Chop House | **Mon open** 12–15, 18–23 | **Mon closed**; 12–14:15, 18–21:45 | ⚠️ stale |
| Rough Trade East | Mo–Sa 10:00–19:00 | Mo–Th 08:00–**21:00** | ⚠️ times stale |

**Headline:** OSM reliably identifies the *place* and the *rough open-days shape*, but
**specific times are frequently stale** — off by an hour or two, occasionally a wrong
closed-day. Treat the data as **indicative, not authoritative**: anything shown to a
user should carry a "hours via OpenStreetMap — check before you travel" note and link
out to the venue.

## Where manual/web research would add the most
Whole categories have **zero** OSM coverage and are worth a hand pass if you want them
populated: **cinemas** (Prince Charles, Phoenix, etc. — 9), **wine bars** (7),
**Polish** spots (15), **archives/livery halls** (8, mostly appointment-only), plus
the **16 missing bakeries** and many famous pubs not yet tagged (Town of Ramsgate,
The Lamb, The Widow's Son…).

## Data format & integration
`data/hours.json`:
```json
{
  "_generated": "ISO timestamp",
  "_source": "OpenStreetMap opening_hours via Overpass (ODbL)",
  "hours": {
    "<spotId>": { "h": "Tu-Su 10:00-18:00", "osm": "node/123", "name": "…", "match": "name", "dist": 6 }
  }
}
```
- `h` is raw **OSM `opening_hours` syntax** (`Mo-Fr 09:00-17:00; Sa 10:00-16:00; Su off`,
  `24/7`, `PH off`…). To render human-readable, parse it (e.g. the `opening_hours.js`
  library) or display verbatim.
- Kept as a **separate file** (your choice): `spots.json` stays untouched. To show
  hours in the app, merge by `id` at build time and add display logic — **not yet
  wired into `build.js`** (data-gathering step only).

## Re-running / refreshing
```bash
node tools/enrich-hours.js --dry --limit 8       # preview
node tools/enrich-hours.js --city london         # full London (resumable)
node tools/enrich-hours.js --city london --force # re-fetch everything (refresh stale)
node tools/enrich-hours.js --cats museum,cinema  # specific categories
```
Resumable (skips ids already resolved unless `--force`), checkpoints every 20 spots,
rotates Overpass mirrors, rate-limited. Re-run periodically to catch OSM updates.
